Class Inclusion with Autoloading
PHP File Inclusion

When I first developed this site, eggmatters.com. I didn't use an autoloader. And now I'm the sorrier for it. I followed an MVC pattern but - any classes I generated had to be manually inserted.

If you know what autoloaders in PHP are and how they work, then you can skip this. Our autoloader will be namespace-based to reduce calls to the filesystem

We can't do this project without some sort of dynamic file inclusion. Because of convention over configuration, users will be generating classes on the fly that the application just needs to know about.

We can't have users, as part of their development process, update all of the core scripts with their own include directives. That would be insane. Think about the amount of technical debt that would generate.

Luckily, PHP provides a mechanism where we can intercept class object instantiations as well as static method calls.

SPL Autoload Register

Luckily, we have an answer. PHP's spl_autoload_register() function kind of acts like an event listener which fires anytime a class instance from our application is put on the stack. Namely, these two invocations will trigger spl_autoload_register(): 

  
    $myFoo = new Foo();
    $myStaticFoo = Foo::staticMethod();

This is a great boon for us, because now we can actually include the resource that defines Foo, only when it is needed.
